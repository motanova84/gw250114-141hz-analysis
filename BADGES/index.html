    Badge Validation¶

    All badges have been validated using scripts/validate_badges.py:

    python3 scripts/validate_badges.py

    Validation Results: - ✅ All workflow files exist - ✅ License file exists - ✅ Documentation files exist - ✅ Python versions are consistent (3.11) - ✅ GWPy version matches requirements - ✅ All badge URLs are properly formatted - ✅ All linked resources are accessible

    Maintenance¶

    When to Update Badges¶

    1. Python Version Badge
    2. Update when minimum Python version changes
    3. Must match all workflow files

    4. Update analyze.yml, production-qcal.yml, and other workflows

    5. GWPy Version Badge

    6. Update when minimum GWPy version in requirements.txt changes

    7. Use version range (e.g., "3.0+") not specific version

    8. CI/CD Badges

    9. Automatically updated by GitHub Actions
    10. No manual intervention needed

    11. Check if workflow file is renamed

    12. DOI Badge

    13. Update when new Zenodo version is released
    14. Format: 10.5281/zenodo.XXXXXXX
    15. Badge URL and link URL must match

    Badge Consistency Checks¶

    Run the validation script regularly:

    # Validate all badges
python3 scripts/validate_badges.py

# Add to CI pipeline
- name: Validate badges
  run: python3 scripts/validate_badges.py

    Badge Best Practices¶

    1. Use Dynamic Badges When Possible
    2. GitHub Actions badges update automatically
    3. Zenodo badges reflect latest version

    4. Shields.io supports many dynamic sources

    5. Link Badges to Resources

    6. Badges should be clickable when relevant

    7. Link to workflow runs, documentation, external sites

    8. Keep Badges Accurate

    9. Version numbers must match actual requirements
    10. Status badges should reflect real state

    11. Remove badges for removed features

    12. Badge Placement

    13. Important badges at top of README
    14. Group related badges together
    15. Don't overcrowd with too many badges

    Troubleshooting¶

    Badge Not Showing¶

    1. Check image URL is accessible
    2. Verify badge syntax: ![Alt](image-url) or [![Alt](image-url)](link-url)
    3. For GitHub Actions badges, ensure workflow file exists
    4. Clear browser cache

    Badge Shows Wrong Status¶

    1. For CI/CD badges: Check workflow run history
    2. For version badges: Verify against source files
    3. For DOI badges: Verify DOI exists on Zenodo

    Badge Link Broken¶

    1. Verify target file/URL exists
    2. Check for typos in file paths
    3. For external links, check domain is accessible
    4. For workflow links, ensure workflow file name matches
